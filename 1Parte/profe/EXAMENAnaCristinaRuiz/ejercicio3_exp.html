Explicación

Este ejercicio consiste en que el usuario, en los "input type radio", seleccionara un color, un tamaño y un radio de la esquinas del cuadrado inferior.

Cuando pulse en la botonera de abajo, o cambiará solo el tamaño, o solo el color, o solo el radio, o bien, cambiará las tres cosas, o bien, lo retornara al estado inicial (50px de ancho/alto, sin radio y sin color (o color blanco).

Estos con diversos estados:

Pistas: Recorre los input(radio) como se ha visto en clase y cuando encuentre el input seleccionado, captura su valor y aplicalo al cuadrado

Entrega el ejercio por ClassRoom. Puedes:

  • Entregar un solo HTML que incluya el script.
  • Si has hecho la parte de JS separado,entregar tanto el HTML como el fichero JS.

Recuerda comentar correctamente el script, así como usar variables con significado. Cuenta para nota ;-)

Rúbrica de Evaluación

Criterio Nivel de Desempeño Descripción Puntos
Funcionalidad Completa Excelente El código cumple completamente la funcionalidad pedida: captura correctamente los inputs y aplica correctamente con los botones los colores, tamaños y radios correpondientes. 1.5
Adecuado El código cumple con la mayoría de la funcionalidad requerida, aunque puede fallar en algún detalle menor o carecer de algo no esencial para el funcionamiento. 1
Necesita Mejora El código tiene errores que afectan su funcionamiento o no cumple con parte de la funcionalidad requerida. 0.5
Insuficiente El código no cumple con la funcionalidad requerida, o presenta errores graves que impiden su ejecución o comprensión. 0
Estructura del Código y Nombres de Variables Excelente El código está bien estructurado en funciones claras y organizadas. Los nombres de variables son descriptivos y coherentes, facilitando la comprensión del código. 0.6
Adecuado El código está estructurado, aunque algunas secciones podrían organizarse mejor. La mayoría de los nombres de variables son claros, pero algunos pueden mejorar. 0.3
Necesita Mejora El código carece de una estructura clara, con funciones o secciones innecesariamente mezcladas, y algunos nombres de variables no son descriptivos. 0.15
Insuficiente El código es confuso y desorganizado, con nombres de variables poco intuitivos, dificultando su comprensión y mantenimiento. 0
Uso de Comentarios Excelente Todos los elementos y secciones del código están comentados de forma clara y concisa, ayudando a entender el propósito de cada parte. 0.4
Adecuado La mayoría de las partes importantes del código están comentadas, aunque algunos comentarios son escasos o genéricos. 0.2
